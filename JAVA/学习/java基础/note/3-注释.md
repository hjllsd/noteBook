## [[注释]]的介绍
用来解释当前注释下的代码或代码块又或者用来注释掉暂时无用或不确定的代码,而我们知道Java的注释分为三种
### 单行注释
#### 单行注释的介绍
单行注释的作用其实无需过多介绍,如果注释的内容或代码在一行内能解决就使用单行注释就行
####  单行注释的语法
``` java
// 这是一个单行注释
```
#### 个人吐槽
个人看来说实话我还真没见过几个人在注释多行代码的时候使用单行注释,毕竟现在都有快捷键了
### 多行注释
#### 多行注释的介绍
多行注释的作用就是需要多行展示即一行写不下的情况下使用
#### 多行注释的语法

``` java
/*
这是多行注释文字
这是多行注释文字
这是多行注释文字
*/
```


> [!NOTE] 注意
> 多行注释不能嵌套使用

### 文档注释
### 文档注释的作用
使用一些**文档注释参数**表明这个类或方法的作用,方法的入参或出参等等,可以**生成文档或者在ide里查看**
###  文档注释的语法
``` java
/**
这是多行注释文字
这是多行注释文字
这是多行注释文字
*/
```

| Javadoc 标签            | 作用描述                          | 示例                                               | 适用对象             |
| --------------------- | ----------------------------- | ------------------------------------------------ | ---------------- |
| `@param <parameter>`  | 描述方法或构造函数的参数                  | `@param username 用户的登录名，需唯一`                     | 方法/构造函数参数        |
| `@return`             | 描述方法的返回值（非void方法）             | `@return 返回操作结果，true表示成功`                        | 非void方法返回值       |
| `@throws <exception>` | 描述方法可能抛出的异常                   | `@throws IOException 当文件读写失败时抛出`                 | 方法可能抛出的异常        |
| `@see`                | 添加相关类/方法的参考链接                 | `@see java.util.ArrayList#add`                   | 类/方法/字段的关联引用     |
| `@deprecated`         | 标记方法/类已过时，建议替代方案              | `@deprecated 使用 {@link #newMethod()} 替代`         | 即将废弃的类/方法        |
| `@author`             | 标识代码作者（通常用于类注释）               | `@author Alan Turing`                            | 类/接口的作者信息        |
| `@version`            | 标识代码版本号                       | `@version 1.8.2`                                 | 类的版本信息           |
| `@since`              | 标记功能引入的版本                     | `@since JDK1.8`                                  | 类/方法的首次引入版本      |
| `{@code <text>}`      | 将文本显示为代码格式（不解析HTML/Javadoc标签） | `{@code List<String> list = new ArrayList<>();}` | 内联代码示例           |
| `{@link <reference>}` | 创建指向类/方法/字段的跳转链接              | `参见 {@link #getUserName()} 方法`                   | 文档内部跳转引用         |
| `{@literal <text>}`   | 原样显示文本（避免<>&等符号被解析）           | `{@literal List<String>} 表示泛型集合`                 | 需保留特殊符号的文本       |
| `@serial`             | 描述序列化字段的说明                    | `@serial 描述这个字段的序列化格式`                           | Serializable类的字段 |
